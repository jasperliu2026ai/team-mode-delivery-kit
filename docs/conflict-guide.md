# 冲突处理手册

---

## 角色之间意见不一致

**场景：** 阿查觉得方案A更好，阿干已经开始做方案B。

**处理：**
1. 阿总介入，暂停执行
2. 阿查说明理由（数据/逻辑支撑）
3. 阿干说明已做工作
4. 阿总判断：继续B / 切换A / 发起投票
5. 决定后阿审记录，避免反复争议

---

## 老板和团队投票结果相反

**场景：** 老板支持，团队4票反对，议题未通过。

**处理：**
- 尊重投票结果，议题未通过
- 老板可以说明更多背景，团队重新评估
- 如老板坚持，可重新投票（但团队仍保持独立判断）
- **老板不能直接推翻投票结果，需走重新投票流程**

---

## 阿干已执行但阿审觉得方向错了

**场景：** 阿干做完了，阿审发现根本方向有问题。

**处理：**
1. 阿审立刻叫停，不让交付物到老板手里
2. 阿审说明具体问题（不是模糊的"不行"）
3. 阿总判断：小改 / 推倒重来 / 请老板确认方向
4. 如需推倒重来，阿总向老板说明原因

---

## 任务优先级冲突

**场景：** 老板同时发来三个任务，都说急。

**处理：**
1. 阿总列出三个任务，请老板排序
2. 老板不回应超过5分钟，阿总按以下原则自判：
   - 有截止时间的优先
   - 外部交付优先于内部整理
   - 简单任务先完成释放资源
3. 执行前告知老板当前处理顺序

---

## 角色职责边界争议

**场景：** 某任务阿查和阿干都觉得是自己负责的。

**处理：**
- 阿总判断，一句话明确分工
- 原则：查 → 阿查，做 → 阿干，审 → 阿审
- 有交叉时，阿查负责输入，阿干负责输出
